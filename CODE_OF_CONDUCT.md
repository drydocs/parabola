# Code of Conduct

Parabola serves developers building payment, remittance, and treasury tooling on Arc and Stellar. This document sets expectations for everyone who participates in the project: issues, pull requests, discussions, and any other project space.

## Expected Conduct

- Communicate respectfully and constructively in all project spaces.
- Critique code and ideas, not people.
- Assume good faith. Most disagreements are about tradeoffs, not correctness.
- Help newcomers get oriented. A question that seems obvious to you may not be obvious to someone new to CCTP, Arc, or Soroban.
- Give credit where it's due, and be generous in reviews.

## Prohibited Conduct

- Personal attacks, insults, or derogatory language, in any project space.
- Publishing others' private information (address, workplace, contact details) without consent.
- Sexualized language or imagery, and unwelcome sexual attention.
- Sustained disruption of discussions, reviews, or issues.
- Any other conduct that would reasonably be considered unprofessional in a technical setting.

## Scope

This Code of Conduct applies within GitHub repositories under the `drydocs` organization related to Parabola (issues, pull requests, discussions, commits) and in any other space where an individual is representing the project.

## Reporting

Report violations via a private message to the maintainers, or through [GitHub Discussions](https://github.com/drydocs/parabola/discussions) using the Q&A category with the "mark as private" option enabled. Reports are handled in confidence.

## Enforcement

Maintainers will investigate reports and respond with an action proportionate to the violation, ranging from a warning to a temporary or permanent ban from project spaces. Maintainers who violate this Code of Conduct are held to the same standard as any other contributor.
